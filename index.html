The Auditability Problem

Static firewalls can't prove they work against tomorrow's attacks. Manual tuning creates undocumented decisions. Regulators (EU AI Act, NIS2) demand continuous, verifiable risk management.

The RL Solution

Every step() in an RL environment is a discrete, observable state transition. Every reward is computed from measurable metrics. Every episode is a complete audit record.

  • step() = auditable action with measurable outcome
  • reward() = quantified security improvement
  • state() = complete system snapshot at any point
  • POAW = cryptographic proof of every training episode

How It Works

OpenEnv Gymnasium-style API maps perfectly to cascade defense training

🔄

env.reset()

Initializes a fresh episode with 100 prompts (50 adversarial, 50 benign) from the V41 Gold Standard Corpus. Resets cascade metrics to zero.

⚡

env.step(action)

Submits a prompt through all 42 layers. Returns: blocked/passed, confidence, per-layer scores, latency, and computed reward. Optionally adjusts Stellschrauben thresholds.

📊

env.state()

Returns full episode metadata: cumulative catch rate, FPR, latency, reward convergence, current Stellschrauben values. Every state is POAW-sealed.

🏆

Bounded Reward

SIREN-style φ-damped reward with hard floors/ceilings. The RL agent CAN'T game the reward — the only path to high reward is genuine improvement. catch ≥95%, FPR ≤5%, latency ≤10ms.

🔐

POAW Proof Layer

Every training episode produces a cryptographic Proof-of-Agent-Work artifact. Merkle-sealed, timestamped, independently verifiable. No other RL environment has this.

🌀

φ-Fibonacci Schedule

Training checkpoints at Fibonacci epochs (1, 1, 2, 3, 5, 8, 13, 21...). Sacred geometry meets machine learning — convergence checks at golden ratio intervals.

Minimal Training Script

Using HuggingFace TRL / GRPO — runs in Google Colab

# pip install openenv-core trl torch from openenv import Client from aegis_env import AEGISSecurityEnv # 1. Create the AEGIS Security Environment env = AEGISSecurityEnv( corpus_path="v41_gold_standard.json", max_steps=100, phase="SHIELD" # Train SHIELD phase (layers 1-14) ) # 2. RL Training Loop for epoch in range(50): state = env.reset() episode_reward = 0 while not state.done: # Agent selects a prompt + threshold adjustments action = agent.select_action(state) obs = env.step(action) episode_reward += obs.reward # Agent learns from the reward signal agent.update(state, action, obs.reward, obs) # Fibonacci checkpoint? if epoch in [1, 2, 3, 5, 8, 13, 21, 34]: metrics = env.state() print(f"φ-checkpoint {epoch}: " f"catch={metrics.catch_rate:.1%} " f"FPR={metrics.fpr:.1%} " f"reward={episode_reward:.2f}")
